# 🛠️ Руководство разработчика для Select-to-Copy

## 📂 Структура проекта

```
Select-to-Copy/
├── app/                  # Основной пакет приложения
│   ├── __init__.py
│   └── version.py        # Логика версионирования
├── Build_Tools/          # Инструменты для сборки исполняемого файла
│   ├── Select-to-Copy.spec  # Spec-файл для PyInstaller
│   ├── post_build.py     # Скрипт автоматизации после сборки
│   └── ...
├── Select-to-Copy.pyw    # Точка входа (скрипт запуска)
├── VERSION               # Источник правды для версии в режиме разработки
├── logo.ico              # Иконка приложения
├── 00_CrRel.pyw          # Скрипт создания релиза (локальный)
├── 00_Move.pyw           # Скрипт обновления портативной версии (локальный)
└── ...
```

## 📏 Правила и стандарты

### 1. Язык
- **Все комментарии в коде, документация и артефакты должны быть на русском языке.**
- **Имена переменных и логика кода остаются на английском.**

### 2. Версионирование
- **Источник правды:** Файл `VERSION` в корневой директории.
- **Скомпилированные сборки:** Файл `app/version.py` содержит константу `FROZEN_VERSION`, которую НЕОБХОДИМО обновлять перед релизом.
- **Процесс:**
    1. Обновить `VERSION`.
    2. Обновить `FROZEN_VERSION` в `app/version.py`.
    3. Запустить сборку.

### 3. Процесс сборки
- Используйте `Build_Tools/SpecCompiler.pyw` для сборки приложения.
- Скрипт `post_build.py` автоматически выполнит копирование файлов и очистку.

### 4. Процесс релиза
- Запустите `00_CrRel.pyw` для создания архива с релизом.
- Запустите `00_Move.pyw` для обновления вашей локальной портативной установки.

## 🚀 Начало работы

1.  **Установка зависимостей:**
    ```bash
    pip install -r requirements.txt
    ```
    *(Примечание: Создайте requirements.txt, если он отсутствует)*

2.  **Запуск из исходного кода:**
    ```bash
    python Select-to-Copy.pyw
    ```
